一、Spring Ioc容器和Beans的介绍(Introduction to the Spring Ioc Container and Beans)

1.Ioc

Ioc(控制反转)也被称为DI(依赖注入)。
它是一个过程，此过程定义了对象之间的依赖关系，即一个对象中需要另一个对象协作的关系。
此关系是通过构造参数，工厂参数或者对象实例中被set的属性建立的。
此过程要在对象实例化之后或者对象由工厂返回之后进行，即对象要先被创建在能定义他们的依赖关系。

2. BeanFactory 和 ApplicationContext

org.springframework.beans 和 org.springframework.context 两个包是Ioc容器的基础。

3. Beans

在Spring中,构成应用骨架的对象或者被Spring Ioc容器管理的对象称为beans.

二、容器简介（Container Overview）

org.springframework.context.ApplicationContext 接口代表了Spring Ioc容器，并且负责实例化beans,配置bean,集成(assemble)。容器通过读取配置元数据(configuration metedata)来获取一些指令，这些指令主要是指明需要实例化哪个对象，配置哪个对象以及集成（assemble）哪个对象。

配置元数据（configuration metedata）的形式有三种，

  • 第一，xml文件形式，
  • 第二，java注解形式，
  • 第三，java代码形式。

配置元数据（configurateion metedata）陈述了对象，这些对象构成了你的应用，同时也陈述了对象之间丰富的相互依赖的关系。

三、Bean简介

Ioc容器管理一个或多个Bean。这些Bean被配置元数据管理(例如被xml文件定义)。在容器本身中，这些bean的定义作为BeanDefinition对象，此类对象包含以下元数据：

  • 全类名（含包结构）
  • Bean行为配置元素
  • 对其他bean的引用
  • 其他配置

这些元数据被转换成一系列的属性，这些属性构成了每个bean的定义。
以下是bean的定义

Property Explain
Class 实例化bean
Name 命名bean
Scope Bean的作用范围
Constructor arguments 构造参数形式的依赖注入
Properties set方式的依赖注入
Autowiring mode 自动装配
Lazy initialzation mode 懒加载
Initialization method 实例化方法
Destruction method 销毁方法

ApplicationContext除了拥有包含如何创建bean的信息，它也允许容器外部的，由用户创建的已经存在的对象被注册。这个功能是由ApplicationContext的BeanFactory通过getBeanFactory方法实现的。它返回DefaultListableBeanFactory的实现方法。但是不太常用。

3.1 Naming Beans (命名bean)

每一个bean都有一个或多个身份。这些身份在容器中必须是独一无二的。一个bean通常只有一个身份。但是如果他需要多个，就需要考虑使用aliases标签。
在基于xml的配置元数据中，你通常使用id标签，name标签，或者两者同时使用来指定一个bean的身份。
id标签指定了一个确切的ID。你也可以通过name标签来命名多个名字，他们通过（,）(;)或者空格相隔开。id标签的独一性是由容器来控制的，xml的语法仅仅定义了id的类型是string。
你不需要强制的为bean提供name标签或者id标签。此时，容器会为bean分配一个唯一的名字。

四、Dependencies(依赖)

一个典型的企业应用不只包含一个单一的对象(bean)。甚至最简单的应用都有一些对象一起协作来呈现最终用户看到的效果。本部分解释了这些bean是符合一起协作的。

4.1 Denpendencies Injection(依赖注入)

依赖注入(DI)是一个过程，此过程中对象定义了他们的依赖(其他一起协作的对象)。
这样代码会更简洁。解耦是更有效率的，当对象和他的依赖一起被提供的时候。
